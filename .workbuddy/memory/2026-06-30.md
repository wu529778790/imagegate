## 样式修复 + 清理

### 问题 1：page.tsx Server Component 导致 CSS Module 无法解析
- 将 page.tsx 从 Server Component 改回 `'use client'`，保留所有 Zustand/SWR/apiClient 重构改进
- CSS Module (`Home.module.css`) 在 Client Component 中正常工作，类名 `Home-module__jKgjFW__layout/main/content` 正确渲染
- 删除冗余 `GenerateShell.tsx`（逻辑已回到 page.tsx）和空的 `features/` 目录

### 问题 2：全局 CSS 未加载
- 根因：`layout.tsx` 改为 Server Component 后，丢失了 `import "./globals.css"` 导入
- 结果：页面背景、文字颜色、导航链接、glass 效果等所有全局样式未应用，页面显示为白色背景和默认蓝色链接
- 修复：在 `src/app/layout.tsx` 中重新添加 `import "./globals.css"`
- 验证：`_next/static/chunks/[root-of-the-server]...css` 中已包含 `--bg-primary`、`.nav-link` 等全局样式
- 需要用户操作：浏览器中执行**强制刷新**（Cmd+Shift+R / Ctrl+F5）以清除缓存

### 经验教训
- **Next.js 全局 CSS 必须在 layout.tsx 或 page.tsx 中显式导入**，仅在组件中导入 CSS Module 不会加载全局样式
- **Server Component 中导入全局 CSS 是允许的**，只是不能导入 CSS Module 跨组件边界
- **页面依赖 CSS Module 时，保持 `'use client'` 指令**，否则样式解析失败

### antd v6 API 修复
- AuthModal.tsx: `maskClosable={!loading}` → `mask={{ closable: !loading }}`, `destroyOnClose` → `destroyOnHidden`

### WASM 路径修复
- sql.js WASM 加载路径从默认 `/ROOT/node_modules/sql.js/dist/sql-wasm.wasm` 改为 `path.join(process.cwd(), "node_modules/sql.js/dist/sql-wasm.wasm")`
- 同时复制 WASM 文件到 `public/sql-wasm.wasm` 作为备用
