## 修复生成图片不绘制 Bug

### 问题
用户反馈：配置 API 后点击生成图片，只请求了 `/api/auth/session`，没有发送 `/api/generate`，也没有显示生成中的状态。

### 根因
`AuthContext.tsx` 中 `requireAuth` 的 `onSuccess` 回调与 `closeAuthModal` 之间存在**竞态条件**：

```typescript
// 修复前（错误顺序）
onSuccess: () => {
  closeAuthModal();           // 内部调用 promiseRef.current(false) → resolve(false)
  promiseRef.current = null;
  resolve(true);              // Promise 已 resolved，被忽略！
}
```

当用户已登录且已配置 API 时：
1. `AuthModal` 打开 → `checkConfig()` 发现已配置 → 调用 `onSuccess()`
2. `onSuccess` 调用 `closeAuthModal()` → 内部 `promiseRef.current(false)` → **resolve(false)**
3. `resolve(true)` 被调用，但 Promise 已 resolved，被忽略
4. 回到 `generate()`：`authed = false` → `if (!authed) return;` → 流程中断

### 修复
将 `promiseRef.current = null` 移到 `closeAuthModal()` **之前**，确保 `closeAuthModal` 不会错误地 resolve(false)：

```typescript
// 修复后（正确顺序）
onSuccess: () => {
  promiseRef.current = null;   // 先清空，防止 closeAuthModal resolve(false)
  closeAuthModal();
  options?.onSuccess?.();
  resolve(true);               // 正常 resolve(true)，生成流程继续
}
```

### 根因
用户配置了 `anthropic_api_key` 但 `default_provider` 为 null：
1. `resetFromSettings` 将 provider 默认为 `"openai"`（硬编码 fallback）
2. 前端发送 `provider: "openai"`，后端查找 `openai_api_key` → 不存在 → 返回 NO_API_KEY

### 修复 (2 处)

**1. 后端 `src/app/api/generate/route.ts`**
- 当请求指定了 provider 但该 provider 无 key 时，不再直接返回 NO_API_KEY
- 新增回退逻辑：扫描所有 VALID_PROVIDERS 在 settings 表和 api_keys 表中查找可用 key
- 同时修复 auto-detect 路径也扫描 settings 表（之前只扫描 api_keys 表）

**2. 前端 `src/stores/generate-store.ts`**
- `resetFromSettings` 新增智能 provider 检测
- 当 `default_provider` 为空时，扫描 settings 中 `{provider}_api_key` 键值
- 自动选择第一个有已配置 key 的 provider（anthropic 或 openai）
